### 1.1 🔴 `routes/loan.ts` — Wrong XDR type for `spender` arg in `approveArgs`

**File:** `backend/src/routes/loan.ts`  
**Location:** `POST /repay` handler, `approveArgs` construction

```typescript
// ❌ CURRENT — toScAddress() returns xdr.ScAddress, not xdr.ScVal
const approveArgs = [
  Address.fromString(wallet).toScVal(),
  Address.fromString(contractIds.lendingPool).toScAddress(), // <-- BUG
  nativeToScVal(totalOwedStroops, { type: "i128" }),
  nativeToScVal(expirationLedger, { type: "u32" }),

// ✅ FIX — use toScVal() so the arg is a proper xdr.ScVal (scvAddress wrapper)
const approveArgs = [
  Address.fromString(wallet).toScVal(),
  Address.fromString(contractIds.lendingPool).toScVal(), // <-- FIXED

  nativeToScVal(expirationLedger, { type: "u32" }),

```

**Impact:** Every repay attempt silently passes a raw `xdr.ScAddress` where a wrapped `xdr.ScVal` is expected. The Soroban RPC rejects the transaction during simulation with a deserialization error, making repayment completely non-functional.

- [x] Change `.toScAddress()` → `.toScVal()` on the lending-pool address argument.

### 1.6 🟡 `routes/admin.ts` — Ad-hoc auth instead of middleware

**File:** `backend/src/routes/admin.ts`

The admin route manually inspects `req.headers.authorization` rather than using a shared middleware. This is inconsistent and bypasses any future improvements to auth.

```typescript
// ❌ CURRENT — manual header check
const authHeader = req.headers.authorization;
if (authHeader !== `Bearer ${config.adminApiSecret}`) {
  throw unauthorized('Admin access only');
}

// ✅ FIX — create an adminAuthMiddleware in middleware/auth.ts
export function adminAuthMiddleware(req: Request, _res: Response, next: NextFunction) {
  const token = req.headers.authorization?.replace(/^Bearer\s+/i, '');
  if (token !== config.adminApiSecret) return next(unauthorized('Admin access only'));
  return next();
}
// Then use it in admin.ts:
router.get('/check-defaults', adminAuthMiddleware, asyncRoute(...));
```

- [x] Add `adminAuthMiddleware` to `backend/src/middleware/auth.ts`.
- [x] Replace the inline header check in `admin.ts` with the new middleware.

## 2. Frontend — `frontend/`

### 2.1 🔴 `next.config.ts` — CSP missing `style-src 'unsafe-inline'`

**File:** `frontend/next.config.ts`

The app uses inline `style={{ ... }}` attributes on virtually every component. Without an explicit `style-src 'unsafe-inline'` directive, browsers fall back to `default-src 'self'`, which **blocks all inline styles**. In production the entire UI renders unstyled.

```typescript
// ❌ CURRENT
`default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval'; connect-src 'self' ...`
// ✅ FIX — add style-src, img-src, and font-src
`default-src 'self'; \
 script-src 'self' 'unsafe-inline' 'unsafe-eval'; \
 style-src 'self' 'unsafe-inline'; \
 img-src 'self' data: https:; \
 font-src 'self' data:; \
 connect-src 'self' ${apiOrigin} ${rpcOrigin} https://*.stellar.org https://stellar.expert https://friendbot.stellar.org`;
```

- [ ] Add `style-src 'self' 'unsafe-inline'` to the CSP header.
- [ ] Add `img-src 'self' data: https:` to prevent broken images (e.g., wallet provider logos).
- [ ] Add `font-src 'self' data:` to prevent broken web-fonts.

### 2.3 🟠 `store/walletStore.ts` — `connect()` accesses `localStorage` without SSR guard

**File:** `frontend/store/walletStore.ts`

`restoreSession` guards `localStorage` access with `typeof window === 'undefined'`, but `connect()` and `disconnect()` call `localStorage` unconditionally. If these are ever triggered server-side (e.g., during pre-rendering), the app will throw `ReferenceError: localStorage is not defined`.

```typescript
// ✅ Add SSR guard to connect() and disconnect()
connect: async () => {
  if (typeof window === 'undefined') return;
  ...
  localStorage.setItem('kredito_wallet_connected', 'true');
},

disconnect: () => {
  if (typeof window !== 'undefined') {
    localStorage.removeItem('kredito_wallet_connected');
  }
  ...
},
```

- [ ] Add `typeof window === 'undefined'` guard before `localStorage` calls in `connect()`.
- [ ] Add the same guard in `disconnect()`.
